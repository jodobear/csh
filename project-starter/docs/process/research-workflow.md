# Research Workflow

Use this workflow when a project begins with research or produces research during execution.

## Rules

- Keep project-created research in `docs/references/local/`.
- Keep imported shared references in `docs/references/imported/`.
- Use `docs/references/local/research-note-template.md` for new notes.
- Keep one note per bounded topic, source family, or decision theme.
- Start with `classification: undecided` unless portability is already clear.
- Update `handoff.md` after meaningful research progress.

## Promotion Rule

- If a note appears reusable outside the project, set its classification and promotion target in
  frontmatter so it can later be reviewed for shared promotion.
