## Project Direction

- This project starts research-heavy.
- ContextVM and Nostr should be treated as primary technical context.
- Keep project docs concise, source-backed, and current.
- The project agent owns updates to local plans, handoff, research notes, and guidance as the work
  evolves.

## Core Rules

- Use the starter process, but adapt it to the real project as quickly as the research justifies.
- Prefer bounded research notes over broad dump documents.
- Keep DM communication summaries in `docs/communications/dm-log.md`.
- Run implementation in small reviewable slices using `docs/process/implementation-loop.md`.
- Update `handoff.md` after meaningful progress and before phase or wave boundaries.
- Record important decisions and tradeoffs in `docs/plans/decision-log.md`.
- If an ambiguity changes architecture, trust boundaries, or remote execution posture, ask a
  targeted clarifying question before locking in the default.
