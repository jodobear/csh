# Prompt-Driven Kickoff

Use this workflow when the operator wants to start from a prompt and references rather than filling
project files manually.

## Rules

- Draft the project profile from the kickoff prompt and references.
- Start the first research pass in `docs/references/local/`.
- Update the build plan and handoff.
- After the first research pass, propose a brief plan and stop for operator review before
  implementation.
- Ask one clarifying question only when a high-impact ambiguity blocks useful progress.
